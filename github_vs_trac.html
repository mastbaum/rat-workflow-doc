git+github vs git+Trac

github git+Trac
Configuration None Find or set up server on site, install git, create accounts for all contributors
Read-only checkout $ git clone http://github.com/cenpa/rat $ git clone http://snolab.ca/rat
Developer Forks Press 'fork' button Users log into server, clone master repo locally with git clone
SCM on forks Issue tracking, repository browser, view of all other forks with ability to pull any changes online None, manually set up an instance of Trac for every repository of every developer, if desired.
Sharing between forks View changes on other forks and pull them in online. Press pull request button to ask for another developer to merge in your changes. Log into server and browse logs of other repositories with git log on the command line. Use git CLI tools (add new remote, fetch, merge) to pull in changes. Email collaborator with a list of commits to ask to have them pulled into another repo.
Code review and merging to "trunk" Click "pull request" button. CIC gets a page with all commits and diffs. If approved, one button merges changes. If not, comments can be made on individial commits, lines of code, or overall. Due to overhead of merging, users push directly to the master repository with (git add remote), git push, and resolve any conflicts.
